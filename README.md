# 셀바이저 가맹점주 APP 모바일 웹 데모

v1.3 Mock 데이터 기반 가맹점주 APP 프로토타입 — 브라우저 또는 모바일 기기에서 바로 확인 가능합니다.

## 바로 실행하기

### 1. 의존성 설치

```bash
npm install
```

### 2. 로컬 개발 서버 실행

```bash
npm run dev
```

콘솔에 두 개의 URL이 표시됩니다.

```
  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/   ← 모바일 기기로 접속
```

### 3. 모바일 기기에서 접속하기 (권장)

**같은 Wi-Fi에 연결된 휴대폰의 브라우저에 Network URL을 입력**하면 실제 모바일 환경에서 확인 가능합니다.

**더 편하게**: Network URL을 QR 코드로 변환 → 휴대폰으로 스캔

```bash
# Mac/Linux
npx qrcode-terminal http://192.168.x.x:5173
```

### 4. iOS 홈 화면에 추가 (PWA처럼 사용)

1. Safari로 Network URL 접속
2. 공유 버튼 → "홈 화면에 추가"
3. 홈 화면의 셀바이저 아이콘 실행 시 주소창·탭바 없이 전체화면 표시

## 데스크탑 접속

`http://localhost:5173/` 접속 시 화면 중앙에 **448px 폭의 모바일 프레임**이 표시됩니다. 크롬 DevTools의 Device Mode (`Ctrl+Shift+M` / `Cmd+Shift+M`)를 사용하면 더 실감나게 확인할 수 있습니다.

권장 기기 프로필:
- iPhone 14 Pro (393 x 852)
- iPhone SE (375 x 667)
- Pixel 7 (412 x 915)

## 프로덕션 빌드 (배포용)

```bash
npm run build      # dist/ 폴더에 정적 파일 생성
npm run preview    # 빌드 결과물을 로컬에서 미리보기
```

`dist/` 폴더를 Vercel · Netlify · Cloudflare Pages 등에 업로드하면 바로 공개 URL로 공유 가능합니다.

### Vercel 배포 (가장 빠름)

```bash
npx vercel         # 최초 1회
npx vercel --prod  # 프로덕션 배포
```

## 데모 계정

Mock 데이터이므로 로그인 불요. 시작 시 자동으로 **브릿지커피 강남점 OWNER (김강남)**로 진입합니다.

역할 전환: 하단 바 `전체` 탭 > 상단 "역할 전환" 드롭다운에서 OWNER / MANAGER / STAFF 선택 가능.

## 기술 스택

- React 18 + Vite 5
- Recharts (차트)
- 인라인 스타일 (CSS/Tailwind 미사용)
- 외부 상태관리 라이브러리 없음 (React useState만 사용)

## 이 데모의 성격

- **데이터**: 100% Mock (하드코딩, 브릿지커피 강남점 시나리오)
- **인증**: 없음 (자동 OWNER 로그인 상태)
- **네트워크 요청**: 없음 (모두 로컬 state 변화)
- **프로덕션 목적이 아닌 영업 시연·검증용**

실제 Supabase 연동 버전은 별도 저장소(`sellvisor-app-preview`)에서 Phase 1으로 진행 예정입니다.

## 문제 해결

**네트워크 URL이 안 뜸**:
- PC의 방화벽이 포트 5173을 차단했을 수 있습니다. 방화벽 허용 필요.

**모바일에서 접속 안 됨**:
- PC와 휴대폰이 같은 Wi-Fi 네트워크에 있는지 확인하세요.
- 회사 네트워크는 장비 간 통신을 차단하는 경우가 있습니다. 핫스팟으로 시도해 보세요.

**차트가 깨짐**:
- 모바일 가로 모드로 회전하면 레이아웃이 깨질 수 있습니다. 세로 모드 권장.

**iOS Safari에서 하단 탭바가 잘림**:
- `viewport-fit=cover` + `100dvh`로 처리했으나 일부 iOS 버전에서 주소창 영역과 겹칠 수 있습니다.
